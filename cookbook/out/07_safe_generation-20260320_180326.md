-- SPL 2.0: Safe Generation with Exception Handling
-- Demonstrates LLM-specific error handling patterns

WORKFLOW safe_generation
  INPUT: @prompt text
  OUTPUT: @result text
  SECURITY: CLASSIFICATION: internal
DO
  GENERATE response(@prompt) INTO @result

  -- Quality check via semantic evaluation (LLM-as-judge)
  GENERATE quality_assess(@result) INTO @quality

  EVALUATE @quality
    WHEN 'high_quality' THEN
      COMMIT @result WITH status = 'high_quality'
    WHEN 'acceptable' THEN
      GENERATE improved(@result, @prompt) INTO @result
      COMMIT @result WITH status = 'refined'
    OTHERWISE
      COMMIT @result WITH status = 'best_effort'
  END

EXCEPTION
  WHEN HallucinationDetected THEN
    GENERATE response(@prompt) INTO @result
    COMMIT @result WITH status = 'conservative'

  WHEN ContextLengthExceeded THEN
    RETRY WITH temperature = 0.1

  WHEN RefusalToAnswer THEN
    COMMIT @prompt WITH status = 'refused'

  WHEN BudgetExceeded THEN
    COMMIT @result WITH status = 'truncated'
END

The internet is a complex network of interconnected computers and servers that allow users to access and share information, communicate with each other, and transfer data. Here's a simplified explanation of how the internet works:

**Basic Components**

1. **Networks**: The internet is made up of many networks, including Internet Service Providers (ISPs), which connect homes, businesses, and organizations to the global network.
2. **Routers**: Routers are devices that direct data packets between networks and ensure they reach their intended destination.
3. **Servers**: Servers store and manage data, such as websites, emails, and online applications.
4. **Clients**: Clients, like web browsers or email clients, request data from servers.

**How Data is Transmitted**

1. **Request**: When you enter a website's URL or send an email, your client (e.g., browser or email client) sends a request to the nearest router on your network.
2. **Routing**: The router forwards your request to its next closest neighbor on the internet, which is also a router.
3. **Packet Switching**: The data packet is divided into smaller parts and transmitted independently through the internet, using a process called packet switching.
4. **Reassembly**: Each router receives a portion of the packet and stores it until all pieces arrive at their destination.
5. **Server Response**: When your request reaches its final destination (the server), the server responds with the requested data.
6. **Return Journey**: The response is sent back through the routers, following the same path as your original request.

**Key Technologies**

1. **IP Addressing**: Each device on the internet has a unique IP address that identifies it and allows data packets to be routed correctly.
2. **HTTP/HTTPS**: Hypertext Transfer Protocol (HTTP) and its secure version (HTTPS) govern how data is transmitted between clients and servers.
3. **Domain Name System (DNS)**: DNS translates domain names (e.g., example.com) into IP addresses, allowing users to access websites easily.

**Additional Aspects**

1. **Caching**: To speed up data transmission, some networks and servers use caching, where frequently accessed data is stored locally.
2. **Content Delivery Networks (CDNs)**: CDNs are geographically distributed networks that cache content at multiple locations, reducing latency for users worldwide.

This simplified explanation should give you a basic understanding of how the internet works. The actual process involves many more complexities and nuances, but this should provide a solid foundation for further exploration!
