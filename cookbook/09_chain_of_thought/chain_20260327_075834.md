-- Recipe 09: Chain of Thought
-- Multi-step reasoning: Research → Analyze → Summarize
-- Each GENERATE feeds into the next via workflow variables.
--
-- Usage:
--   spl run cookbook/09_chain_of_thought/chain.spl --adapter ollama -m gemma3 topic="distributed AI inference"
--   spl run cookbook/09_chain_of_thought/chain.spl --adapter ollama -m llama3.2 topic="quantum computing"

WORKFLOW chain_of_thought
    INPUT: @topic TEXT
    OUTPUT: @summary TEXT
DO
    -- Step 1: Research
    GENERATE research(@topic) INTO @research

    -- Step 2: Analyze
    GENERATE analyze(@research) INTO @analysis

    -- Step 3: Summarize
    GENERATE summarize(@analysis) INTO @summary

    COMMIT @summary WITH status = 'complete'
END

**Distributed AI Inference Techniques**

1.  **Data Parallelism**: Divide the dataset into smaller chunks that need to be processed independently before combining results.
2.  **Model Parallelism**: Split the model across multiple machines and train each part of the model in parallel.
3.  **Pipelining**: Apply a series of operations on input data in sequence, often with different parts performing different types of computations.

**Challenges in Distributed AI Inference**

1.  **Communication Overhead**: There is usually some overhead involved in sending information between machines, which can slow down computation.
2.  **Synchronization Challenges**: Ensuring that all nodes are working on the same model and have access to the necessary data can be challenging.
3.  **Fault Tolerance**: Distributed systems can be more susceptible to failures due to the nature of their distributed design.
