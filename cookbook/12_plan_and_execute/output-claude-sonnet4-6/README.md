# Todo REST API

A simple REST API for managing todo items, built with FastAPI, SQLAlchemy 2.0, and SQLite.
Generated by [SPL Recipe 12: Plan and Execute](../readme.md), then reviewed and fixed by Claude Code CLI.

## Stack

- **FastAPI** — web framework with automatic OpenAPI docs
- **SQLAlchemy 2.0** — ORM with `Mapped` / `mapped_column` style
- **SQLite** — default database (swap via `DATABASE_URL` env var)
- **Alembic** — database migrations
- **Pydantic v2** — request/response schema validation
- **Uvicorn** — ASGI server

## Project layout

```
output/
├── main.py          # FastAPI app, routes, Pydantic schemas
├── database.py      # Engine, session factory, shared Base
├── config.py        # Settings loaded from environment variables
├── models/
│   ├── __init__.py
│   └── todo.py      # Todo ORM model
├── alembic/
│   ├── env.py       # Alembic migration environment
│   └── versions/    # Migration scripts (empty — generated by alembic)
├── alembic.ini      # Alembic config
├── requirements.txt # Loose version pins for pip
└── pyproject.toml   # Pinned deps + dev extras (pytest, httpx)
```

## Quick start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Run the server (creates todos.db automatically on first start)
uvicorn main:app --reload

# 3. Open interactive API docs
open http://localhost:8000/docs
```

## API endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/todos` | Create a todo |
| `GET` | `/todos` | List all todos (optional `?completed=true/false`) |
| `GET` | `/todos/{id}` | Get a single todo |
| `PATCH` | `/todos/{id}` | Update title and/or completed status |
| `DELETE` | `/todos/{id}` | Delete a todo |

## Example requests

```bash
# Create
curl -X POST http://localhost:8000/todos \
  -H "Content-Type: application/json" \
  -d '{"title": "Buy milk"}'

# List all
curl http://localhost:8000/todos

# List incomplete only
curl "http://localhost:8000/todos?completed=false"

# Mark complete
curl -X PATCH http://localhost:8000/todos/1 \
  -H "Content-Type: application/json" \
  -d '{"completed": true}'

# Delete
curl -X DELETE http://localhost:8000/todos/1
```

## Configuration

| Env var | Default | Description |
|---------|---------|-------------|
| `DATABASE_URL` | `sqlite:///./todos.db` | SQLAlchemy database URL |
| `DB_ECHO` | `false` | Log all SQL statements (set to `true` for debugging) |

## Using Alembic migrations (optional)

By default the app calls `Base.metadata.create_all()` on startup, which is fine for development.
For production, use Alembic:

```bash
# Generate a migration from model changes
alembic revision --autogenerate -m "initial"

# Apply migrations
alembic upgrade head
```

## Post-generation fixes (applied by Claude Code CLI)

The code was generated by `claude-sonnet-4-6` via SPL Recipe 12 and contained 5 bugs that prevented it from running. These were identified and fixed by Claude Code CLI — a realistic example of the generate → review → fix workflow.

| # | File | Bug | Fix |
|---|------|-----|-----|
| 1 | `database.py` | `from config import settings` — `settings` variable doesn't exist in `config.py` | Changed to `from config import get_settings; settings = get_settings()` |
| 2 | `database.py` | `settings.DATABASE_URL` — wrong attribute name (case mismatch) | Changed to `settings.database_url` to match the `Settings` class definition |
| 3 | `models/todo.py` | Defined its own `Base(DeclarativeBase)` — a second, separate Base that Alembic and `init_db()` never see | Removed the local `Base`; import `Base` from `database` so all models share one metadata registry |
| 4 | `main.py` | Completely self-contained — duplicated engine, session, and model definitions; ignored `database.py`, `config.py`, and `models/todo.py` entirely | Rewrote to import from the other modules; upgraded from deprecated `declarative_base()` to SQLAlchemy 2.0 `select()` style |
| 5 | — | Missing `models/__init__.py` and `alembic/versions/` directory | Created both; without `__init__.py` the `from models import todo` import in `alembic/env.py` and `database.py` would fail |

Additionally, `models/todo.py` used old-style `Column` declarations with plain type annotations (`id: int = Column(...)`) which are incompatible with SQLAlchemy 2.0's `Mapped` style. Updated to `Mapped[int] = mapped_column(...)` and replaced the deprecated `datetime.utcnow` with `datetime.now(timezone.utc)`.
