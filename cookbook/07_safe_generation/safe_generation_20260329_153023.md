WORKFLOW safe_generation
  INPUT: @prompt text
  OUTPUT: @result text
  SECURITY: CLASSIFICATION: internal
DO
  GENERATE response(@prompt) INTO @result

  -- Quality check via semantic evaluation (LLM-as-judge)
  GENERATE quality_assess(@result) INTO @quality

  EVALUATE @quality
    WHEN 'high_quality' THEN
      COMMIT @result WITH status = 'high_quality'

      GENERATE improved(@result, @prompt) INTO @result
      COMMIT @result WITH status = 'refined'
    ELSE
      COMMIT @result WITH status = 'best_effort'
  END

EXCEPTION
  WHEN HallucinationDetected THEN
    GENERATE response(@prompt) INTO @result
    COMMIT @result WITH status = 'conservative'

  WHEN ContextLengthExceeded THEN
    RETRY WITH temperature = 0.1 LIMIT 3

  WHEN RefusalToAnswer THEN
    COMMIT @prompt WITH status = 'refused'

  WHEN BudgetExceeded THEN
    COMMIT @result WITH status = 'truncated'
END

Encryption is the process of converting plaintext (readable data) into ciphertext (unreadable data) to ensure confidentiality, integrity, and authenticity. Here's a step-by-step explanation of how encryption works:

1. **Key Generation**: The encryption algorithm generates a secret key, which is used to encrypt and decrypt the data. The key can be a random number or a combination of numbers that are used to scramble the data.
2. **Data Preparation**: The plaintext data is prepared for encryption by dividing it into blocks or chunks, depending on the encryption algorithm used.
3. **Encryption Algorithm**: The encryption algorithm uses the secret key to perform a mathematical operation on each block of data. This operation can be a substitution (e.g., replacing each letter with a different letter), permutation (shifting and rearranging the letters), or combination (using both substitution and permutation).
4. **Ciphertext Generation**: After applying the encryption algorithm to each block of data, the resulting ciphertext is generated.
5. **Data Transmission**: The encrypted data (ciphertext) is transmitted over an insecure channel, such as the internet.

**Encryption Techniques**:

There are two main types of encryption techniques:

1. **Symmetric Encryption**: Uses the same secret key for both encryption and decryption.
2. **Asymmetric Encryption**: Uses a pair of keys: one public key for encryption and a private key for decryption.

**Common Encryption Algorithms**:

Some popular encryption algorithms include:

1. **AES (Advanced Encryption Standard)**: A symmetric encryption algorithm widely used for data protection.
2. **RSA (Rivest-Shamir-Adleman)**: An asymmetric encryption algorithm commonly used for secure online transactions.
3. **Blowfish**: A symmetric encryption algorithm known for its fast processing speed.

**Decryption**:

To decrypt the ciphertext, the receiving party uses the secret key or private key to apply the inverse operation of the encryption algorithm. This returns the original plaintext data.

In summary, encryption works by using a secret key to scramble plaintext data into unreadable ciphertext, ensuring confidentiality, integrity, and authenticity.
