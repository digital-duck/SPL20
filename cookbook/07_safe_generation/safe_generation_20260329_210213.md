WORKFLOW safe_generation
  INPUT: @prompt text
  OUTPUT: @result text
  SECURITY: CLASSIFICATION: internal
DO
  GENERATE response(@prompt) INTO @result

  -- Quality check via semantic evaluation (LLM-as-judge)
  GENERATE quality_assess(@result) INTO @quality

  EVALUATE @quality
    WHEN 'high_quality' THEN
      COMMIT @result WITH status = 'high_quality'
    WHEN 'acceptable' THEN
      GENERATE improved(@result, @prompt) INTO @result
      COMMIT @result WITH status = 'refined'
    ELSE
      COMMIT @result WITH status = 'best_effort'
  END

EXCEPTION
  WHEN HallucinationDetected THEN
    GENERATE response(@prompt) INTO @result
    COMMIT @result WITH status = 'conservative'

  WHEN ContextLengthExceeded THEN
    RETRY WITH temperature = 0.1 LIMIT 3

  WHEN RefusalToAnswer THEN
    COMMIT @prompt WITH status = 'refused'

  WHEN BudgetExceeded THEN
    COMMIT @result WITH status = 'truncated'
END

Encryption is the process of converting plaintext (readable data) into ciphertext (unreadable data) to ensure confidentiality and security. Here's a simplified explanation of how encryption works:

**The Encryption Process**

1. **Key Generation**: A key pair is created, consisting of a public key and a private key. The public key is used for encrypting data, while the private key is used for decrypting data.
2. **Encryption Algorithm**: An encryption algorithm (e.g., AES, RSA) is applied to the plaintext data using the public key. This algorithm transforms the data into ciphertext.
3. **Ciphertext Generation**: The encrypted data is generated and stored.

**Decryption Process**

1. **Plaintext Data**: The sender sends the ciphertext to the intended recipient.
2. **Key Exchange**: The recipient uses their private key to decrypt the ciphertext, but first, they need to exchange keys with the sender using a secure channel (e.g., SSL/TLS).
3. **Decryption Algorithm**: The encryption algorithm is applied to the ciphertext data using the private key. This reverses the transformation done during encryption.
4. **Plaintext Recovery**: The original plaintext data is recovered, making it readable by authorized parties only.

**Key Concepts**

* **Symmetric Key**: A secret key shared between both parties (sender and recipient) for encrypting and decrypting data.
* **Asymmetric Key**: A pair of keys, where the public key is used for encryption and the private key is used for decryption.

**Real-World Applications**

Encryption is used in various applications:

* Secure online transactions (e.g., PayPal)
* Email encryption
* Virtual Private Networks (VPNs)
* Cloud storage
* Secure communication protocols (e.g., HTTPS)

In summary, encryption is a crucial security measure that ensures confidentiality and integrity of data by transforming it into unreadable ciphertext using an encryption algorithm and a key pair.
