-- SPL 2.0: Safe Generation with Exception Handling
-- Demonstrates LLM-specific error handling patterns

WORKFLOW safe_generation
  INPUT: @prompt text
  OUTPUT: @result text
  SECURITY: CLASSIFICATION: internal
DO
  GENERATE response(@prompt) INTO @result

  -- Quality check via semantic evaluation (LLM-as-judge)
  GENERATE quality_assess(@result) INTO @quality

  EVALUATE @quality
    WHEN 'high_quality' THEN
      COMMIT @result WITH status = 'high_quality'
    WHEN 'acceptable' THEN
      GENERATE improved(@result, @prompt) INTO @result
      COMMIT @result WITH status = 'refined'
    ELSE
      COMMIT @result WITH status = 'best_effort'
  END

EXCEPTION
  WHEN HallucinationDetected THEN
    GENERATE response(@prompt) INTO @result
    COMMIT @result WITH status = 'conservative'

  WHEN ContextLengthExceeded THEN
    RETRY WITH temperature = 0.1 LIMIT 3

  WHEN RefusalToAnswer THEN
    COMMIT @prompt WITH status = 'refused'

  WHEN BudgetExceeded THEN
    COMMIT @result WITH status = 'truncated'
END

Encryption is the process of converting plaintext (readable data) into ciphertext (unreadable data) to protect it from unauthorized access. Here's a simplified explanation of how encryption works:

**The Encryption Process**

1. **Key Generation**: A pair of keys is created: a public key and a private key. The public key is used for encrypting data, while the private key is used for decrypting data.
2. **Encryption**: When data needs to be encrypted, it's fed into an encryption algorithm along with the public key. The algorithm uses the public key to scramble the data, converting it into ciphertext.
3. **Ciphertext**: The resulting ciphertext is unreadable without the corresponding private key.

**Types of Encryption**

1. **Symmetric-key encryption**: Uses the same secret key for both encryption and decryption, such as AES (Advanced Encryption Standard).
2. **Asymmetric-key encryption**: Uses a pair of keys: public and private keys, like RSA (Rivest-Shamir-Adleman).

**How Encryption Works**

When data is encrypted:

1. The encryption algorithm takes in plaintext data and the public key.
2. It applies a mathematical formula to the data using the public key, producing ciphertext.
3. The resulting ciphertext is stored or transmitted.

To decrypt the data:

1. The recipient uses the private key to decrypt the ciphertext.
2. The decryption algorithm applies an inverse mathematical formula to the ciphertext using the private key.
3. The resulting plaintext data is readable and usable.

**Real-World Applications**

Encryption is used in various applications, including:

* Secure online transactions
* Email encryption
* Virtual private networks (VPNs)
* Data-at-rest protection

In summary, encryption works by converting plaintext into unreadable ciphertext using a mathematical algorithm and a pair of keys. The recipient uses the corresponding key to decrypt the data, restoring it to its original readable form.
